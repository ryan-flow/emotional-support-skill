# Emotional Support Skill

给AI Agent用的心理疏导框架。

做这个东西的原因很简单：大部分AI在面对情绪问题的时候，要么灌鸡汤，要么列一堆"你可以试试XXX"的建议清单，要么直接说"建议你去看心理咨询师"。这些都没什么用。

这个skill提供一套结构化的方法，让AI在陪伴用户的时候知道该怎么做，什么时候该听，什么时候该问，什么时候该闭嘴。

## 里面有什么

四个心理学框架，每个解决一个不同的问题：

Socratic提问——不给答案，用问题帮对方自己想清楚。来自古典哲学，现在广泛用在心理咨询里。

CBT认知三角——焦虑不是因为事情本身，而是因为你对事情的解读。认知行为疗法的核心。帮你识别"我什么都做不好""ta一定觉得我很差"这类自动化想法。

人本主义倾听——Carl Rogers那套。反映、正常化、情感标注。有时候人不需要建议，只需要被听见。

ACT接纳承诺——Steven Hayes提出的。你不需要消灭焦虑才能行动。带着焦虑也能往前走。

## 适用场景

用户带着这些情绪来的时候触发：焦虑、迷茫、自我否定、选择困难、渴望亲密关系、社交媒体比较焦虑、对未来不确定。

## 怎么用

Hermes Agent用户：放到 `~/.hermes/skills/wellbeing/emotional-support/SKILL.md`

其他AI Agent：把SKILL.md的内容作为system prompt的一部分注入。

## 会话顺序

倾听反映 → 情感标注 → 正常化 → Socratic提问 → 最小行动建议

不是每次都要走完。有时候走到"正常化"就够了。

## 原则

先听后问再建议。不评判。不比惨。不急着正能量。承认不确定性。
